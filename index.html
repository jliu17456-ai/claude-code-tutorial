🌱 小白级

这是什么神器？

想象一下：你坐在终端前，用大白话说出想做的事—— "帮我修这个 bug"、"给这个项目加个登录功能"、"解释下这段代码"—— 然后一个超级聪明的 AI 程序员真的读你的代码、写代码、跑测试、提交 PR。 这就是 Claude Code。

🧠

它会读你的整个代码库

不用你复制粘贴。它自己搜索文件、理解结构、找到相关代码，像一个刚入职但学得飞快的同事。

🛠️

它真的会动手

编辑文件、运行命令、跑测试、查 Git 历史、调用工具。不只是"建议"，而是真正帮你完成。

💬

就在你的终端里

不用切换窗口、不用插件。也能在 VS Code / JetBrains、桌面 App、网页版（claude.ai/code）里用。

🤖

可以无限定制

斜杠命令、钩子、子代理、MCP……你能把它打造成专属于你团队工作流的智能体。

💡 一句话理解
Claude Code = 一个住在你终端里、能读能写能跑代码的 AI 结对程序员。你负责说想法，它负责把活干漂亮。

五分钟安装启动

三步上车。挑一种你喜欢的方式。

1

安装

最快的方式是用官方安装脚本（macOS / Linux）：

终端
curl -fsSL https://claude.ai/install.sh | bash

或者用 npm（需要 Node.js 18+）：

终端
npm install -g @anthropic-ai/claude-code
2

进入项目并启动

切到你的项目目录，敲一个词：

终端
cd my-project
claude

第一次运行会让你登录（浏览器授权或粘贴 API Key）。用 /login 随时切换账号。

3

体检一下

启动后输入 /doctor 检查安装是否健康，再用 /help 看看能干啥。搞定！

claude
/doctor
/help
💡 不想装也能玩
打开 claude.ai/code 即可在网页里直接用 Claude Code，连接你的 GitHub 仓库，零安装上手。

你的第一句对话

别紧张，就像跟人聊天一样。下面是新手最常用的几句"咒语"。

「这个项目是做什么的？带我快速过一遍结构」
→ 它会探索代码库并给你一份导览
「src/auth.js 里的登录逻辑有什么问题？」
→ 它会读文件、定位 bug、解释原因
「帮我加一个深色模式切换按钮」
→ 它会改代码、可能还会跑测试验证
「把刚才的改动提交，写个清晰的 commit message」
→ 它会 git add / commit（征得你同意后）
⚠️ 三条黄金心法
  • 说清楚目标：与其说"优化代码"，不如说"把这个函数的循环改成 map，并加上类型注解"。
  • 大事拆小步：复杂任务先让它做计划（见 Plan 模式），再动手。
  • 你是船长：它每次危险操作前会问你；用 Esc 随时打断、纠正方向。
🚀 进阶级

四大核心概念

理解这四个词，你就从"会用"迈向"用好"。

📜

上下文（Context）

Claude "记得"的对话内容是有容量的。对话太长它会自动压缩（compact）。用 /context 查看占用，用 /clear 开新局，保持它"脑子清醒"。

🧩

工具（Tools）

读文件、写文件、跑 bash、搜索代码……都是"工具"。它会自主选择用哪个。权限模式决定哪些工具需要你点头同意。

🧠

记忆（CLAUDE.md）

项目根目录的 CLAUDE.md 是它的"常驻记忆"——你的编码规范、架构说明、常用命令都写在这。每次都会自动加载。

🔐

权限模式（Permissions）

从"每步都问"到"自动放行"。用 /permissions 配置，或启动时加 --permission-mode。安全与效率的平衡阀。

🚀 进阶级

CLAUDE.md：项目的大脑

这是最能提升效果的一招。一个好的 CLAUDE.md 能让 Claude 立刻"懂你的项目"，少走弯路。 用 /init 可以一键自动生成初版。

CLAUDE.md
# 项目：电商后台

## 技术栈
- Next.js 14 (App Router) + TypeScript
- Prisma + PostgreSQL
- Tailwind CSS

## 常用命令
- 开发：`pnpm dev`
- 测试：`pnpm test`（改完代码务必跑一遍）
- 类型检查：`pnpm typecheck`

## 编码规范
- 组件用函数式 + 具名导出
- 禁止使用 any；优先用 zod 做运行时校验
- 提交信息遵循 Conventional Commits

## 注意事项
- 数据库迁移必须经过 review，不要直接改 schema
💡 三级记忆
  • ./CLAUDE.md — 项目级，团队共享（提交进 Git）
  • ./CLAUDE.local.md — 个人本地（写进 .gitignore）
  • ~/.claude/CLAUDE.md — 全局，所有项目生效
随时用 /memory 编辑，或在对话里用 # 开头快速追加一条记忆。

Plan 模式

按 Shift+Tab 切换到"只规划不动手"模式。让它先给出步骤方案，你点头后再执行——复杂任务的救命稻草。

⏪

后悔药

改坏了？用 /rewind 回退到之前的对话/代码检查点。Esc 两下可编辑历史消息、重开分支。

🧹

勤清上下文

切换到不相关的新任务时，先 /clear。上下文干净 = 回答更准、更省钱。

🖼️

贴图与拖文件

直接把截图粘进终端，或拖入文件路径。报错截图、设计稿、日志——它都能看。

斜杠命令是什么？

在输入框开头敲一个 /，就会弹出一串命令菜单。它们是 Claude Code 的"快捷指令"—— 管理会话、配置、Git、记忆、子代理……玩转斜杠命令，就掌握了 Claude Code 的方向盘。

💡 怎么用
输入 / 后开始打字会模糊匹配命令名，按 Tab 或回车选中。不确定有哪些？直接 /help。

🧙 大神级

自定义你的斜杠命令

内置命令不够用？自己造！在 .claude/commands/ 放一个 Markdown 文件，文件名就是命令名。 这是把团队"套路"固化成一键操作的杀手锏。

.claude/commands/review-pr.md
---
description: 审查当前分支的改动并给出修改建议
argument-hint: [关注点，可选]
allowed-tools: Bash(git*), Read, Grep
---

请审查当前分支相对 main 的全部改动：

1. 用 `git diff main...HEAD` 查看变更
2. 重点关注：$ARGUMENTS
3. 找出潜在 bug、安全问题与可读性问题
4. 用清单形式给出可执行的修改建议

之后在对话里输入 /review-pr 性能 即可触发，$ARGUMENTS 会被替换成 "性能"。

💡 进阶技巧
  • 放在 ~/.claude/commands/ 则是全局命令，所有项目可用。
  • 用子目录可以创建命名空间，如 .claude/commands/git/sync.md → /git:sync。
  • 正文里可用 !`命令` 嵌入 shell 输出，用 @文件 引用文件内容。
  • 用 /agents 和 /hooks 可进一步把命令、子代理、钩子串成自动化流水线。
🧙 大神级

钩子 Hooks：让它自动养成好习惯

钩子是在特定时机自动执行的脚本。比如"每次编辑完文件就自动跑格式化"、"提交前自动跑测试"。 由 Claude Code 的引擎执行，而不是靠 Claude 自觉——所以 100% 可靠。

用 /hooks 交互式配置，或写进 .claude/settings.json：

.claude/settings.json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          { "type": "command", "command": "pnpm prettier --write $CLAUDE_FILE_PATHS" }
        ]
      }
    ]
  }
}
💡 常见钩子时机
PreToolUse（工具调用前，可拦截）、PostToolUse（调用后）、UserPromptSubmit（你发消息时）、Stop（回答结束时）、SessionStart（会话开始，适合装依赖）。
🧙 大神级

MCP：给 Claude 接上外部世界

MCP（Model Context Protocol）是一套标准接口，让 Claude Code 连接外部工具和数据源—— GitHub、数据库、Slack、Figma、浏览器自动化……瞬间多出一堆新工具。

终端 / 添加一个 MCP 服务器
# 添加一个本地 MCP 服务器
claude mcp add my-server -- npx -y @some/mcp-server

# 在会话内查看 / 管理已连接的服务器
/mcp
💡 用 /mcp 干什么
查看已连接服务器、它们提供的工具、认证状态（OAuth 登录），以及临时启用/禁用某个服务器。MCP 工具通常名为 mcp__服务器名__工具名。
🧙 大神级

子代理军团：一个人指挥一支队伍

子代理（Subagents）是拥有独立上下文的专职 AI。你可以让一个"探索代理"去翻遍代码库、 让一个"测试代理"去跑验证，主对话保持干净。这是大神级并行作战的核心。

用 /agents 创建和管理子代理。每个子代理有自己的系统提示、可用工具和模型。

🔭

探索代理

专门做大范围代码搜索，只把结论带回来，不污染主上下文。

📐

架构代理

专门设计实现方案，权衡取舍，输出分步计划。

🔍

审查代理

专门 review 代码，揪 bug、提简化建议。

💡 并行加速
多个互不依赖的子代理可以同时跑。把一个大任务拆给几个代理并行处理，总时长大幅缩短。
🧙 大神级

大神十二诀

把这些刻进肌肉记忆，你就是团队里那个"Claude Code 用得最骚"的人。

01

先计划再动手。复杂任务永远先用 Plan 模式产出方案。

02

CLAUDE.md 是 1 号优先级。把规范写进去，胜过反复叮嘱。

03

勤用 /clear。换任务就清场，准确率与省钱双赢。

04

把套路做成斜杠命令。重复三次的操作就该自动化。

05

用钩子守底线。格式化、测试、lint 交给钩子自动跑。

06

子代理做脏活。大搜索、长验证丢给子代理，主线保持清爽。

07

给它"看"的能力。贴报错截图、贴日志、贴设计稿。

08

Esc 是你的方向盘。跑偏立刻打断，别等它跑完。

09

/rewind 不怕翻车。大胆尝试，随时回到检查点。

10

盯住 /context 和 /cost。知道脑容量和花费，心里有数。

11

接 MCP 扩展边界。让它直连你的真实工具与数据。

12

小步快跑、频繁提交。每个小成果都 commit，方便回溯。

常见问题

Claude Code 收费吗？

需要 Anthropic 账户（Claude 订阅或 API 额度）。用 /cost 看本次会话花费，用 /usage 看用量。

它会偷偷改我的代码或删文件吗？

默认情况下，写文件、跑命令等"危险"操作都会先征求你同意。权限模式可调，但你始终是船长。删除/覆盖前它会先确认。

对话太长变慢/变笨怎么办？

用 /compact 压缩，或 /clear 开新局。换任务务必清场。

能离开终端用吗？

能。VS Code / JetBrains 插件、桌面 App（Mac/Win）、网页版 claude.ai/code 都可以。

怎么报 bug 或给反馈？

会话里输入 /bug 或 /feedback，直接把问题反馈给团队。

我用的是哪个模型？怎么换？

用 /model 切换（如 Opus / Sonnet / Haiku）。最强是 Claude 4.X 系列。
